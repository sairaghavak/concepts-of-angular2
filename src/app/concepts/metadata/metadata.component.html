MetaData

  • Needless to say that MetaData is all about data about data.
  • In the current programming world, metadata is widely used, following the convention over configuration thereby reducing the verbosity of xml and other external configuration files.
  • In any given angular project, we mostly see files with a .ts extension i.e., a typescript file.
  • We refer to some of the typescript file as modules, components, services and pipes etc. It is because of the metadata that helps angular to recognize these typsecript files as modules, components etc.
  • We encapsulate the metadata using a decorator. As said in the Modules section, I repeat that these decorators are similar to annotations in java. Example: @Override.
